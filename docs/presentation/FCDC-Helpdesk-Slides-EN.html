Best Practices — DO's ✅

  • Always submit a ticket for all IT concerns
  • Provide complete details in the description
  • Attach a screenshot when there is an error
  • Choose the correct category
  • Use the appropriate priority level
  • Reply in ticket conversation — not via Viber
  • Submit feedback after resolution
  • Check notifications regularly
  • Protect your password — never share it
  • One ticket per issue

Common Mistakes — DON'Ts ❌

  • Message IT personally instead of submitting a ticket
  • Submit a ticket with no description
  • Use Critical priority for minor issues
  • Create duplicate tickets for the same issue
  • Ignore follow-up messages from IT
  • Share login credentials
  • Submit tickets for colleagues (teach them instead)
  • Rely on verbal requests only
