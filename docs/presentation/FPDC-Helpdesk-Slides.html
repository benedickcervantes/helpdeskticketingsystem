Learning Objectives

Sa pagtatapos ng seminar, dapat kayong makakapag:

  • Mag-login at ma-access ang inyong dashboard
  • Mag-submit ng IT support ticket nang tama at kumpleto
  • Pumili ng tamang category at priority
  • I-track ang status ng ticket (Open → In Progress → Resolved)
  • Makipag-usap sa IT team gamit ang ticket conversation
  • Magbigay ng feedback pagkatapos ma-resolve ang issue

Bakit Kailangan Natin Ito?

❌ Old Way

Text/Viber/Messenger sa IT personally
Walang record kung sino ang nag-request
Nawawala ang request sa chat
Hindi alam kung sino ang nag-aasikaso
Walang priority — lahat "urgent"

✅ Helpdesk System

Formal ticket na may ticket number
May audit trail at history
Naka-save at ma-track ang status
May assigned IT staff
May tamang priority level at feedback

Benefits Para sa Inyo

⚡ Mas Mabilis na Support

Automatic routing sa tamang IT staff. Hindi na kayo maghahanap kung sino ang available.

Company account, encrypted login, role-based access.

Good vs Bad Ticket Example

💡 Pro tip: Mag-attach ng screenshot ng error — i-tap ang Attach Screenshot area sa Create Ticket form

❌ Bad Example

Title: Problem
Description: Ayaw gumana
Category: Other
Priority: Critical

❌ Walang screenshot · kulang ang detalye

✅ Good Example

Title: Outlook hindi makapag-send — Billing

Description:
• Simula 9 AM hindi makapag-send
• Error: "Cannot connect to server"
• Restart ko na ang Outlook
• PC-12, Maria Santos

Category: Email | Priority: High

📎 Attachment: screenshot ng error message

Best Practices — DO's ✅

  • Laging mag-ticket para sa lahat ng IT concerns
  • Magbigay ng complete details sa description
  • Mag-attach ng screenshot kung may error
  • Piliin ang tamang category
  • Gumamit ng tamang priority
  • Sumagot sa ticket conversation — hindi sa Viber
  • Mag-feedback pag na-resolve na
  • I-check ang notifications regularly
  • I-protect ang password — huwag i-share
  • Isang ticket per issue

Common Mistakes — DON'Ts ❌

  • Mag-message sa IT personally sa halip na mag-ticket
  • Mag-submit ng ticket na walang description
  • Gamitin ang Critical para sa minor issues
  • Mag-create ng duplicate tickets
  • I-ignore ang follow-up messages ng IT
  • Mag-share ng login credentials
  • Mag-ticket para sa colleague (turuan sila)
  • Umasa lang sa verbal request
